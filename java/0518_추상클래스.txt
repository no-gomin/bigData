## 추상클래스(abstract) : 추상적으로 정의할테니(선언해 놓을테니) 객체 사용자가 꼭 재정의(overriding) 하시오.

추상메소드가 하나 이상 있을시, 반드시 추상클래스가 되어야 함.

상속받는 클래스가 반드시 재정의(overriding) 하도록 하기 위해 사용함.

public abstract void kimchi();    // 선언만...

final : 반대대는 개념임. 
-> 클래스 앞 : 상속금지                     
-> 멤버 메소드 앞 : 오버라이딩 금지    
-> 멤버변수 앞 : 상수화(=변경금지)
